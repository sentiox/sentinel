# Sentinel — Domain Routing Service for OpenWrt

## Установка Sentinel

Полная инструкция доступна в документации проекта.

### Кратко

Для установки и обновления используйте один скрипт:

```sh
sh <(wget -O - https://raw.githubusercontent.com/sentiox/sentinel/main/install.sh)
```


## Важно перед установкой

- Проект находится в **стадии бета** и активно развивается. Поведение и конфигурация могут меняться от версии к версии.
- При возникновении проблем требуется **технически корректный фидбэк** (логи, описание действий, версия OpenWrt).
- При обновлении **обязательно очищайте кэш LuCI** (очистка кэша браузера или жёсткая перезагрузка страницы).
- После обновления **всегда проверяйте конфигурацию Sentinel** — структура настроек может измениться.
- Требуется **не менее 25 МБ свободного места** во flash-памяти роутера.  
  Устройства с 16 МБ flash **не поддерживаются**.
- При запуске Sentinel **временно модифицирует конфигурацию dnsmasq**.
- Sentinel **изменяет конфигурацию sing-box**.  
  Если используется собственный конфиг sing-box — **сохраните его перед установкой**.
- Информация в этом файле может устаревать. Актуальные изменения публикуются в Telegram-чате проекта.
- Поддерживается **только OpenWrt версии 24.10**.

## Совместимость и ограничения

### Несовместимые пакеты

Перед установкой Sentinel **обязательно удалите**, если они присутствуют:

- `https-dns-proxy`
- `nextdns`
- `luci-app-passwall`
- `luci-app-passwall2`
- `getdomains`

### Удаление getdomains

Если установлен `getdomains`, его необходимо удалить перед установкой Sentinel.  
После удаления рекомендуется перезагрузить роутер.

---

## Диагностика

Если Sentinel работает некорректно:

- проверьте логи Sentinel и sing-box
- убедитесь, что конфигурация dnsmasq восстановилась корректно
- проверьте наличие свободного места
- убедитесь, что используется OpenWrt 24.10
